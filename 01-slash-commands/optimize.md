# /optimize

Analyze the current codebase for performance, readability, and maintainability improvements.

## Steps

1. **Profile** — Identify slow or inefficient code paths
2. **Simplify** — Find overcomplicated logic that can be streamlined
3. **Report** — List findings with severity (CRITICAL / HIGH / MEDIUM / LOW)

## Guidelines

- Focus on real bottlenecks, not micro-optimizations
- Prefer measurable improvements over speculative ones
- Suggest concrete changes with before/after examples
- Respect the existing code style and conventions
